## Copywriting Contract

### Primary CTAs

| Element | Copy | SF Symbol |
|---------|------|-----------|
| Toolbar import entry | "+" (icon-only button that opens a `Menu`) | `plus` |
| Menu item ŌĆö Scan | "Scan Document" | `camera` (leading) |
| Menu item ŌĆö Photos | "Choose from Photos" | `photo.on.rectangle` (leading) |
| Menu item ŌĆö Files | "Import from Files" | `folder` (leading) |
| Viewer close | "Close" (accessibility label only; icon is `xmark`) | `xmark` |
| Rename alert title | "Rename Document" | ŌĆö |
| Rename alert Save button | "Save" | ŌĆö |
| Rename alert Cancel button | "Cancel" | ŌĆö |

**Toolbar "+" button accessibility:** `.accessibilityLabel("Add Document")` ŌĆö required; button renders a `plus` symbol with no visible text label.

### Empty State ŌĆö Document List

Shown when the trip's `documents` relationship is empty. Centered in the List area.

| Element | Copy |
|---------|------|
| Icon | SF Symbol `doc.text` at `.font(.system(size: 56))`, `.secondary` color |
| Heading | "No Documents Yet" (`.title2` semibold, `.primary`) |
| Body | "Tap + to scan, pick a photo, or import a file." (`.subheadline`, `.secondary`) |

**No inline CTA button.** Per CONTEXT.md D15 the empty state is text-only and points to the existing "+" toolbar button ŌĆö do not render a duplicate button in the empty state body. 32pt gap below icon, 8pt between heading and body (matches Phase 1 empty state rhythm).

### Document List ŌĆö Row Copy

| Row element | Copy / Source |
|-------------|---------------|
| Leading icon | `doc.richtext` if `kind == .pdf`; `photo` if `kind == .image`. `.secondary` color, `.body` size |
| Primary label | `Document.displayName` (`.body`, `.primary`) |
| Secondary label | Formatted `importedAt` using `.dateTime.year().month().day()` ŌåÆ e.g. "Apr 19, 2026" (`.subheadline`, `.secondary`) |
| Trailing accessory | Disclosure chevron (`chevron.right`) ŌĆö standard `NavigationLink`-style affordance hinting tap opens viewer |

Rows use the default SwiftUI `List` grouped-inset style ŌĆö no custom row backgrounds.

### Import Auto-Naming (reference, from D13)

The view layer does not prompt for a name on import. The importer generates:

| Source | `displayName` format |
|--------|----------------------|
| Scan | "Scan {date}" where {date} is `Date.now` formatted `.dateTime.year().month().day()` ŌåÆ e.g. "Scan Apr 19, 2026" |
| Photos | "Photo {date}" ŌåÆ e.g. "Photo Apr 19, 2026" |
| Files | Source URL `deletingPathExtension().lastPathComponent` ŌĆö whatever the user's filename was, extension stripped |

The name is shown in the list immediately after the background file-copy Task resolves (D18). A progress spinner (SwiftUI `ProgressView().controlSize(.small)`) replaces the "+" icon while the import is in-flight; it reverts to `plus` on completion or failure.

### Viewer Top Chrome

Minimal. No share button, no overflow menu, no bottom toolbar.

| Element | Copy / Rendering |
|---------|------------------|
| Leading button | `xmark` SF Symbol in a 44├Ś44 tappable area; `.accentColor`; `.accessibilityLabel("Close")` |
| Centered title | `Document.displayName` rendered as `.title2` semibold, `.primary`; truncates with middle ellipsis if wider than available space; single-line |
| Trailing | empty (reserved for future v2 share; do NOT render a placeholder in v1) |

Chrome background: `.ultraThinMaterial` behind an `HStack`, extending into the safe-area top. Content area below the chrome is pure `.systemBackground` (PDF/image fills it).

### Error States

| Situation | Copy | Presentation |
|-----------|------|--------------|
| Import fails (file copy error, scan cancelled-with-error, PhotosPicker transferable load fail) | "Couldn't add document. Please try again." | SwiftUI `.alert` on `DocumentListView`, single "OK" button |
| Viewer fails to load file (file missing on disk) | "This document is unavailable." | Replaces the PDFView/Image body inside the viewer; `X` close remains available. SF Symbol `exclamationmark.triangle` above the copy, `.secondary` color |
| Rename save fails (SwiftData error) | "Couldn't rename. Please try again." | `.alert` after the rename alert dismisses |
| Delete fails after file removal (SwiftData error) | "Couldn't delete. Please try again." | `.alert` on `DocumentListView`. File cleanup failure alone does NOT surface an alert ŌĆö it is logged only (per D16: "throws caught and logged, but does not block model delete") |

**Note:** The "file missing" error copy is deliberately terse. Phase 2 does not attempt auto-recovery; if the file is gone the row should still be deletable via long-press context menu.

### Destructive Actions

Phase 2 adds exactly one new destructive surface: Delete Document.

| Action | Trigger | Confirmation |
|--------|---------|--------------|
| Delete Document | Long-press on document row ŌåÆ `.contextMenu` ŌåÆ "Delete" (destructive role, `trash` SF Symbol) | `.confirmationDialog` with title **"Delete \"{displayName}\"?"**, message **"This removes the file from your device and cannot be undone."**, buttons: "Delete" (destructive role) + "Cancel" (cancel role) |
| Rename Document | Long-press on document row ŌåÆ `.contextMenu` ŌåÆ "Rename" (`pencil` SF Symbol) | `.alert` with `TextField` prefilled with current `displayName`; Save (default) + Cancel (cancel role). Empty trimmed input disables Save |

**Long-press context menu is the ONLY action surface for rename + delete.** Per D15 there is no swipe-to-delete on document rows ŌĆö this diverges intentionally from Trip's swipe pattern and must not be "corrected" during execution. Document rows open the viewer on tap; long-press opens the context menu.

**Cascade delete copy (Trip delete):** The existing Phase 1 Trip delete `.confirmationDialog` already states "This will also delete all documents, packing items, and activities for this trip." No change needed in Phase 2 ŌĆö the copy already covers Document cascade.

### TripDetail Documents Card ŌĆö Wire-Up Copy

The Phase 1 placeholder card at `SectionCard` is wired in Phase 2.

| State | Title | Body |
|-------|-------|------|
| Zero documents | "Documents" | "No documents yet" (`.subheadline`, `.secondary`) |
| Ōēź1 document | "Documents" | Two-line body: first line = "{count} document" / "{count} documents" (pluralized; `.subheadline`, `.secondary`); second line = latest document's `displayName`, truncated to single line (`.subheadline`, `.primary`) |

SF Symbol remains `doc.text` (Phase 1 lock) in the card header ŌĆö do not switch to `doc.richtext` here. The header icon represents the section, not a document kind.

Tap target: the entire card area is tappable. Tap ŌåÆ push `AppDestination.documentList(trip.persistentModelID)`. Do not use a child `Button` ŌĆö apply `.contentShape(Rectangle())` + `.onTapGesture` or wrap in `NavigationLink(value:)`.

### Sheet & Cover Dismiss Behavior

- Import sheets (Scan, Photos, Files) dismiss automatically on user cancel or successful source selection. The background file-copy Task continues after dismissal; the row appears in the list when the Task hops back to `@MainActor` and inserts the Document (D18). The list's `@Query` observes the change.
- `.fullScreenCover` (viewer) dismisses only via the "X" button ŌĆö swipe-to-dismiss is disabled by using `.fullScreenCover` (which has no default swipe-dismiss) to avoid the gesture competing with pinch-zoom (D14).
- `.alert` (rename) dismisses on Save or Cancel; swipe is not available on alerts.
- `.confirmationDialog` (delete) dismisses on Delete, Cancel, or tap outside.

### Validation Rules

| Field | Rule | Feedback |
|-------|------|----------|
| Rename ŌĆö new name | Trimmed whitespace; must be non-empty | Save button disabled while trimmed draft is empty. No inline error text. |
| Rename ŌĆö new name | No character restrictions (emoji, punctuation allowed ŌĆö it is only a display label, not a filename) | ŌĆö |
| Rename ŌĆö max length | Soft cap at 120 characters ŌĆö `TextField` does not enforce; truncation in list via SwiftUI's native `.lineLimit(1)` + truncation | ŌĆö |
| Import ŌĆö file type | Whitelist: `UTType.pdf`, `UTType.image` (covers JPEG, PNG, HEIC) | Enforced at picker level; user cannot select unsupported types |
| Import ŌĆö file size | No explicit cap in v1 | ŌĆö |

## Phase 2 Locked Decisions (Additive to Phase 1)

These decisions are made here and must not be revisited without a design contract amendment:

| Decision | Value | Rationale |
