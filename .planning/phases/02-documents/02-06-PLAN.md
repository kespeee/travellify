## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test runner → FileStorage | Tests invoke real Documents-directory writes; must not escape sandbox or leak state between test runs |
| test fixtures → test bundle | Binary fixtures shipped in test target resources are trusted (we generated them); not an attacker-controlled input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06-01 | Tampering | Test suite leaking Documents/ state between runs (polluting subsequent runs or production app's Documents folder on simulator) | mitigate | Every test uses `defer { try? FileStorage.removeTripFolder(tripIDString: tripID) }` with a unique UUID per test; no hard-coded paths |
| T-02-06-02 | Information Disclosure | Test fixtures unintentionally containing PII or real user data | accept | Fixtures are synthetic minimal files (1-frame, solid-color) generated in Task 1; no user data present |
| T-02-06-03 | Denial of Service | Test suite runtime exceeding 45s Nyquist budget | mitigate | In-memory ModelContainer (no disk SQLite); tiny fixtures (<50KB each); no sleeps; parallel execution by default in Swift Testing |
| T-02-06-04 | Tampering | Schema drift later introducing `@Attribute(.unique)` or `.deny` breaking future CloudKit sync | mitigate | `noUniqueOrDenyInModels` @Test runs on every `xcodebuild test` — fails CI before the bad schema ships |
| T-02-06-05 | Spoofing | Test asserts byte-equality against the wrong fixture (e.g., resolves the source file instead of the destination) | mitigate | `filesImportCopiesToDestination` asserts destination path matches `^[A-F0-9-]+/[A-F0-9-]+\\.pdf$` — rejects any path that retains the source filename, catching T-02-03-like path regressions |
